## OpenClaw运行方式

### 1. 系统架构
```
B仔的OpenClaw环境：
├─ 核心服务（24/7运行）
│  ├─ OpenClaw Gateway（网关服务）
│  ├─ 任务调度器（定时触发）
│  ├─ 消息路由器（通信枢纽）
│  └─ 数据持久化（SQLite/JSON）
│
├─ Agent系统（模块化运行）
│  ├─ B仔主agent（你正在对话的）
│  ├─ JDdog-agent-LP（刚开发的模块）
│  ├─ 裁判agent（评估模块）
│  ├─ 管家agent（交付模块）
│  └─ 执行agent（浏览器自动化）
│
└─ 工作空间（专用数据目录）
   ├─ /home/wangzhen-bot/hunting_data/（专用存储）
   ├─ 配置文件（身份/规则/偏好）
   ├─ 数据库文件（候选人/任务）
   ├─ 日志文件（执行记录）
   └─ 输出文件（报告/表格）
```

### 2. 运行形式：混合模式
```
核心服务：24小时运行（任务调度/数据存储）
Agent会话：按需启动（具体任务执行）

结合优势：
- 定时任务自动执行（如每天搜索）
- 即时任务快速响应（如新需求）
- 数据持续积累（候选人库）
- 消息即时交互（微信/飞书）
```

### 3. 工作流程示例
```
【每天08:00】定时任务触发
↓
B仔启动JDdog-agent-LP
↓
读取最新客户需求
↓
生成25个关键词组合
↓
调用执行agent进行猎聘搜索
↓
初筛候选人，存入数据库
↓
调用裁判agent进行评估
↓
通过评估的打包给管家agent
↓
【09:00】管家agent发消息给你：
"桢哥，今日搜索完成，找到5个匹配候选人，详情见附件"
↓
你查看Excel表格，给出下一步指令
↓
管家agent安排面试、跟进进度
```

### 4. 数据安全方案
```
专用存储目录：/home/wangzhen-bot/hunting_data/
├─ 物理隔离：猎头数据与其他文件分离
├─ 访问控制：B仔只能访问此目录
├─ 权限明确：你知道所有数据位置
├─ 备份简单：备份整个目录即可
├─ 清理方便：删除目录即清除所有数据
└─ 监控容易：检查目录变化即可
```

### 5. 技术配置示例
```yaml
# OpenClaw配置示例
agents:
  b_main:           # B仔主agent
    model: deepseek-chat
    workspace: /home/wangzhen-bot/hunting_data/
    skills: [猎头自动化]
    
  jddog_agent:      # JD解析agent  
    model: deepseek-chat
    skills: [jd_parser, keyword_generator]
    
  judge_agent:      # 裁判agent
    model: deepseek-chat  
    skills: [candidate_evaluation]
    
  butler_agent:     # 管家agent
    model: deepseek-chat
    skills: [report_generation, process_management]

scheduler:
  daily_search:
    cron: "0 8 * * *"      # 每天8点
    agent: b_main
    task: "启动今日猎聘搜索"
    
storage:
  database: sqlite:///home/wangzhen-bot/hunting_data/candidates.db
  workspace: /home/wangzhen-bot/hunting_data
```

## 系统特点

### 1. 自动化程度
```
✅ 全流程自动化：从需求接收到结果交付
✅ 智能解析：JD自动解析为搜索条件
✅ 平台适配：精确匹配招聘平台规则
✅ 质量导向：不以数量为标准，关注匹配质量
```

### 2. 安全性
```
✅ 数据隔离：专用存储目录
✅ 访问控制：B仔权限限制
✅ 加密存储：敏感信息加密
✅ 审计日志：所有操作可追溯
```

### 3. 可扩展性
```
✅ 模块化设计：各agent独立可替换
✅ 多平台支持：猎聘、BOSS、智联等
✅ 规则可配置：评估规则可调整
✅ 学习优化：基于效果自动优化
```

### 4. 用户体验
```
✅ 消息驱动：通过微信/飞书交互
✅ 结果可视化：Excel表格+报告
✅ 进度透明：随时查看执行状态
✅ 指令响应：自然语言指令控制
```

## 使用方式

```
作为用户（桢哥），你只需：
1. 发消息：把JD和需求发给B仔
2. 自动化：系统自动执行搜索、评估、打包
3. 收结果：收到Excel表格和消息通知
4. 下指令：回复下一步操作指令

就像：
- 你有一个24小时在线的猎头助理
- 助理背后有一个完整的自动化系统
- 系统在OpenClaw环境中运行
- 数据存在专用目录里
- 通过消息与你交互
```
