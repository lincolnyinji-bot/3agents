# 🐝 小蜜蜂Agent架构设计

## 🎯 设计目标
**智能JD理解 + 精准搜索 + 双层初筛 + 记忆去重**

## 🏗️ 系统架构

### **核心模块**：
```
1. 🧠 JD理解器 (JDParser)
   └── 大模型驱动，JD文本 → 结构化搜索策略

2. 🔍 智能搜索器 (SmartSearcher)  
   └── 猎聘平台适配，执行精准搜索

3. 🎯 双层初筛器 (DoubleScreener)
   ├── 严格快筛（早上）：硬性规则过滤
   └── 宽松快筛（晚上）：补漏放宽条件

4. 💾 记忆系统 (CandidateMemory)
   └── 避免重复处理，记录历史

5. 📊 结果处理器 (ResultProcessor)
   └── 结构化输出，对接裁判Agent
```

## 🔄 工作流程

### **完整流程**：
```
1. 输入JD文本
2. JD理解器 → 搜索策略
3. 智能搜索器执行搜索
4. 获取候选人列表
5. 双层初筛器过滤
6. 记忆系统去重
7. 输出给裁判Agent
8. 记录处理历史
```

### **定时任务**：
```
早上 09:00：严格快筛模式
晚上 20:00：宽松快筛模式
```

## 📊 数据结构

### **搜索策略**：
```python
@dataclass
class SearchStrategy:
    keywords: List[str]          # 搜索关键词
    filters: Dict[str, Any]     # 筛选条件
    sort_by: str                # 排序方式
    max_results: int = 100      # 最大结果数
```

### **候选人卡片**：
```python
@dataclass  
class CandidateCard:
    id: str                     # 候选人ID
    name: str                   # 姓名
    current_position: str       # 当前职位
    current_company: str        # 当前公司
    education: str              # 教育背景
    experience: str             # 工作经验
    location: str               # 所在地
    # ... 其他字段
```

### **初筛结果**：
```python
@dataclass
class ScreeningResult:
    candidate_id: str
    screening_type: str         # "strict" | "loose"
    result: str                 # "pass" | "reject" | "need_detail"
    reason: str                 # 原因说明
    timestamp: datetime
```
