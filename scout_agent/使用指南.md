# 🐝 小蜜蜂Agent使用指南

## 🎯 概述
小蜜蜂Agent是一个智能候选人发现系统，能够：
1. **解析JD要求**（大模型驱动）
2. **智能搜索**猎聘平台
3. **双层初筛**（严格/宽松模式）
4. **记忆去重**避免重复处理
5. **对接裁判**评估通过候选人

## 📁 文件结构
```
scout_agent/
├── scout_agent.py      # 主程序
├── config.yaml         # 配置文件
```

## 🔧 核心功能

### **1. JD处理**
```python
from scout_agent import ScoutAgent, ScreeningType

# 初始化Agent
agent = ScoutAgent(use_model=True)  # 使用大模型解析JD

# 处理JD
jd_text = "寻找深圳的AI方向投资经理，3-5年经验，985/211优先"
result = agent.process_jd(jd_text, ScreeningType.STRICT)

print(f"找到 {result['passed_candidates']} 个通过初筛的候选人")
```

### **2. 定时任务**
```python
# 每日严格快筛（早上9点）
results = agent.daily_strict_screening(jd_texts)

# 每日宽松快筛（晚上8点）  
results = agent.daily_loose_screening(jd_texts)
```

### **3. 状态监控**
```python
status = agent.get_status()
print(f"已处理: {status['stats']['total_candidates']} 个候选人")
print(f"通过: {status['stats']['passed_candidates']} 个")
```

## 🎛️ 配置说明

### **初筛规则配置**
编辑 `config.yaml` 中的 `screening` 部分：

```yaml
strict_rules:
  - name: "学校要求"
    field: "education"
    condition: "contains_not: ['985', 'QS']"
    action: "reject"
```

### **支持的字段**：
- `education`: 教育背景
- `current_position`: 当前职位  
- `experience`: 工作经验
- `location`: 所在地
- `expected_city`: 期望城市

### **支持的条件**：
- `contains`: 包含文本
- `contains_not`: 不包含文本
- `contains_any`: 包含任意一个
- `contains_all`: 包含所有
- `less_than`: 小于数值
- `greater_than`: 大于数值
- `always_true`: 总是为真

### **支持的动作**：
- `pass`: 通过初筛
- `reject`: 拒绝
- `need_detail`: 需要查看详情

## 🔄 对接裁判Agent

### **自动对接**：
在 `config.yaml` 中配置：
```yaml
output:
  judge_integration:
    enabled: true
    endpoint: "http://裁判Agent地址/evaluate"
```

### **手动对接**：
```python
# 获取通过初筛的候选人
result = agent.process_jd(jd_text, ScreeningType.STRICT)
passed_candidates = result['passed_list']

# 发送给裁判Agent评估
for candidate_data in passed_candidates:
    candidate = candidate_data['candidate']
    # 调用裁判API
    evaluation = judge_agent.evaluate(candidate)
```

## 🐛 故障排除

### **常见问题**：

1. **登录失败**
   ```
   检查：config.yaml中的账号密码
   解决：确认账号有效，网络可访问猎聘
   ```

2. **搜索无结果**
   ```
   检查：搜索关键词是否合理
   解决：调整config.yaml中的搜索参数
   ```

3. **初筛规则不生效**
   ```
   检查：规则条件是否正确
   解决：查看日志，调整规则配置
   ```

4. **内存占用过高**
   ```
   检查：candidate_memory.json文件大小
   解决：调整memory.cleanup_days配置
   ```


### **减少API调用**：

## 🔐 安全建议

### **账号安全**：
1. 使用专用猎聘账号
2. 定期更换密码
3. 不在代码中硬编码密码

### **数据保护**：
1. `candidate_memory.json` 包含候选人信息
2. 建议加密存储或限制访问
3. 定期清理敏感数据

### **合规使用**：
1. 遵守猎聘平台使用条款
2. 合理控制请求频率
3. 仅用于合法招聘目的
